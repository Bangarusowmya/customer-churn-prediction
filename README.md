# Customer Churn Prediction

A production-ready ML pipeline for predicting customer churn in a telecom company. Built as a portfolio project — covers everything from raw data to a deployed Streamlit app.

---

## Problem Statement

Telecom companies lose 15–25% of their customer base every year. Acquiring a new customer costs 5–10× more than retaining an existing one, so predicting *who is about to leave* — before they do — is genuinely valuable.

This project answers: **"Given a customer's account details and usage patterns, how likely are they to cancel in the next month?"**

---

## Dataset

**IBM Watson Analytics Telco Customer Churn Dataset**

- 7,043 customers, 20 features
- Target variable: `Churn` (Yes/No) — 26.5% churn rate (moderately imbalanced)
- Features include demographics, account details, and subscribed services

| Column | Description |
|---|---|
| `tenure` | How many months they've been a customer |
| `MonthlyCharges` | What they pay each month |
| `Contract` | Month-to-month, 1-year, or 2-year |
| `InternetService` | DSL, Fiber optic, or No |
| `TotalCharges` | Cumulative billing amount |
| `Churn` | **Target** — did they cancel? |

---

## Approach

1. **EDA** — Identify key patterns (contract type and tenure are the biggest drivers)
2. **Feature Engineering** — Added 3 features with clear business rationale
3. **Preprocessing** — Median imputation, StandardScaler for numerics, OneHotEncoder for categoricals
4. **Modeling** — Trained Logistic Regression, Random Forest, and XGBoost; selected best by ROC-AUC
5. **Evaluation** — Confusion matrix, ROC curve, feature importance
6. **Deployment** — Streamlit app for real-time predictions

---

## Feature Engineering

| Feature | Formula | Reasoning |
|---|---|---|
| `charges_per_tenure` | `MonthlyCharges / (tenure + 1)` | High cost relative to tenure = risk of buyer's remorse |
| `is_high_value` | `MonthlyCharges > $65` (binary) | Higher-paying customers churn more in this dataset |
| `num_services` | Count of 6 add-on services | More services = more locked-in to the ecosystem |

---

## Results

| Model | Accuracy | Precision | Recall | F1 | AUC |
|---|---|---|---|---|---|
| Logistic Regression | ~80% | ~65% | ~54% | ~59% | ~0.84 |
| Random Forest | ~80% | ~65% | ~49% | ~56% | ~0.83 |
| **XGBoost** | **~80%** | **~65%** | **~55%** | **~60%** | **~0.85** |

**Best model: XGBoost** — chosen for highest ROC-AUC. Accuracy alone is misleading with a 73/27 class split.

**Key business insight:** Contract type and tenure dominate predictions. A customer on a month-to-month plan in their first 12 months is in the highest risk category.

## How to Run

### 1. Clone and install dependencies

```bash
git clone https://github.com/Bangarusowmya/customer-churn-project.git
cd customer-churn-project
pip install -r requirements.txt
```

### 2. Train the model

```bash
python main.py
```

This will:
- Load `data/raw.csv`
- Run feature engineering
- Train 3 models and compare them
- Save the best model to `models/churn_model.pkl`
- Save evaluation charts to `models/`

### 3. Launch the Streamlit app

```bash
streamlit run app/app.py
``
`## 🚀 Live Demo
Open the app in your browser:
 `https://customer-churn-prediction-7e4rrwmwlqhtmguwdjvfux.streamlit.app/` .

### 4. (Optional) Run the EDA notebook

```bash
jupyter notebook notebooks/eda.ipynb
```
